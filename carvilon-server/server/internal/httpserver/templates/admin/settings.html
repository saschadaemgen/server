Shelly Integration
Shelly enabled Shows the configured Shelly devices as their own category in the Device Center (read only, no switching) and turns on mDNS auto-discovery. Without any configured device the default is off.
Manual IP addresses Only needed as a fallback for devices in another subnet or VLAN, where multicast discovery cannot reach them. Gen2+ devices on the same LAN are found automatically. LAN IPv4 addresses, comma-separated, e.g. 192.168.1.51, 192.168.1.52. Saving an empty list removes the manual pins (discovered devices are kept).
Auth password Only needed when the devices are password protected (digest authentication, user "admin"). Stored AES-256-GCM encrypted and never shown again - enter a new value to replace it.
Connection Live reachability of the configured devices.

Auto-discovery Gen2+ Shelly devices on the local network are found automatically over mDNS.{{if .Page.Shelly.DiscoveredCount}} Currently {{.Page.Shelly.DiscoveredCount}} discovered device{{if ne .Page.Shelly.DiscoveredCount 1}}s{{end}} active.{{end}} Removed devices stay removed - release them below to allow re-discovery.
Scan network Actively sweep your local network for Shelly devices - this finds devices that hide from mDNS (some ship with discovery off), so you never need to type an IP. Only the harmless identify endpoint is probed, never outside your own private subnet. Found devices appear under Pending approval below.
Auto-activate discovered devices Off (recommended): a found device waits for your approval below and is not contacted until then - safe on any network. On: found devices become active immediately (only recommended on a segmented VLAN you trust). This affects new finds only; devices already pending are left as they are.
Keep Shelly cloud When a device is approved it is provisioned onto the CARVILON MQTT broker (TLS) and hardened. Off (recommended): the device's Shelly-cloud connection is turned off. On: it is left enabled (Gen2+ can run the cloud and our broker in parallel). Applies at the next provisioning.
Pending approval Devices found on the network waiting for your approval. They are not contacted or polled until you approve them, so a pending row shows only what the announcement carried (MAC + address). Approve to start polling it, or reject to keep it out.
